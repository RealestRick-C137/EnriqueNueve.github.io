Python implementation of the paper "A General Reflex Fuzzy Min-Max Neural Network."

July 2019

Intro

  A common issue that arises in creating models is deciding between high accuracy and low interpretability or high interpretability and low accuracy. This issue becomes even cloudier when the data doesn't fully represent the events that are being modeled. Due to the lack of information provided in the data, a compromise that allows for the continual use of the data is weakening the restrictions of the accepted outputs of the model. A modern implementation of this type of model is a Bayesian Neural Network. This model has weights that consist of two trainable parameters that represent mean and variance. This allows the model to output a range of possible outcomes. Although Bayesian Networks and Bayesian Deep Learning overall are currently growing, there exists another family of models that allow for high levels of interpretability. These models are called Fuzzy Networks.

Fuzzy Networks

  Fuzzy Networks are based around the theorems of fuzzy sets. Fuzzy sets were made by Lotfi A. Zadeh and Dieter Klaua in 1965. A fuzzy set consists of a set and a membership function, which states how much an element(data-sample) is part of that particular fuzzy set. These ideas of fuzzy sets were further developed by Zadeh into the concept of fuzzy logic. Unlike classical logic, fuzzy logic allows for continuous-intervals of truth. The continuous truth intervals allow for an element to be given a non-binary value from zero to one rating of how much that element is part of a particular fuzzy set. Fuzzy sets also have very clean geometric properties allowing for easy visual interpretation of how a particular element relates to a certain fuzzy set. Through the rules of fuzzy logic, Fuzzy Networks were inspired.

  Fuzzy networks are usually classification models; however, there are variants that do regression. In this post, I will be covering Fuzzy Networks that do classification. Furthermore, Fuzzy Networks are supervised, learning models. The data set consists of training examples with labeled targets of two or more different classes. Fuzzy Networks create trainable sets based around fuzzy sets called hyperboxes. Hyperboxes consist of two parameters, a min, and a max, that define a lower and upper corner of a rectangular area. Although I am describing the hyperboxes having a rectangular area based on the min and max parameters, and this is only for the sake of conceptual explanation. Hyperbox min and max parameters can be vectors larges than two dimensions. Two dimensions simply allow for a convenient visual explanation for the system. All the principles of the Fuzzy Network apply to N-dimension data samples. Hyperboxes also have a membership function that states how much an element is part of that particular hyperbox based on its distance from the defined hyperboxe's area.

The membership function takes in several variables. The function takes the min~V and max~V parameter of the hyperbox. It takes in a tuning parameter gamma (gamma values vary from one to four on average), which adjusts the sensitivity of the distance measurement. The membership function has a composed function within it, a ramp function, which sets a tested element membership value to one of the elements falls directly within its area.

Finally, beyond the membership function, the hyperbox has a tag saying which class it is associated with. Fuzzy Networks, during their training period, have two stages: expansion and contraction.

  During the expansion phase, hyperboxes are created or expanded by connecting to other hyperboxes. When a data sample is introduced during the expansion phase; two outcomes can happen. One, a new hyperbox could be formed. This occurs when no hyperboxes meet expansion criteria (refer to the paper for further detail); thus a new hyperbox of the respective labeled class is created. This implies whenever a data sample from a class is introduced for the first time, a hyperbox with the tag of that respective class will always be made. The min and max parameters will be a vector equivalent to the training sample's values. It is necessary that your input data be scaled between zero and one. Initially, min and max parameters are equal to each other, although this set is called a hyperbox, it initially, geometrically appears as a point. The second event that can occur during the expansion phase is expansion itself.

  If two points meet the expansion criteria, then two hyperboxes join together to form one large hyperbox with min and max parameters based on the min[min-parameter] and max[max-parameter] of the two sets. Expansion can only occur if the class of the two hyperboxes is the same. The second part of the training phase, contraction, occurs if the hyperbox that was just expanded now overlaps an already existing hyperbox of different classes. Based off of different cases in how the boxes overlap, the overlapping boxes get assigned new min and max parameters, causing the boxes to be no longer overlapping.

  After the training period in which hyperboxes are made, for the different data classes presented in the training data, the model is now ready to predict test data. When the model is shown a test sample, the membership value is calculated for every hyperbox that was created. Of these calculated membership values, the max membership values from any particular defined class are considered the prediction.

  This is traditionally how Fuzzy Networks work, however, in the paper "A General Reflex Fuzzy Min-Max Neural Network" by Abhijeet V. Nandedkar and Prabir Kumar Biswas, the idea of a Fuzzy Network was built further to create a new network called the General Reflex Fuzzy Min-Max Neural Network [1].

General Reflex Fuzzy Min-Max Neural Network

  The General Reflex Fuzzy Min-Max Neural Network, GRFMMN, is unique in the way in which it adds two new types of hyperboxes that help break ties between samples that are closely categorized. In total, there are three types of hyperboxes in the GRFMMN: classifying, overlap compensation, and containment compensation hyperboxes. The classifying hyperbox is the same hyperbox in the standard Fuzzy Network, so I will be discussing the two unique hyperboxes in the GFRMMN, which are the overlap compensation neuron (OCN) and the containment compensation neuron (CCN).

  The overlap compensation neuron is created during the contraction phase of the Fuzzy Network's training. When the network contracts two overlapping hyperboxes, a new hyperbox, the overlap compensation neuron, is formed with the min and max values being assigned to the overlapping rectangular area of the two initial hyperboxes.

  Like the standard hyperbox, the OCN has its own unique membership function expressing how much an element is part of that particular hyperbox. The unique thing about the OCN is that it belongs to both classes that were overlapping. Although trivial-sounding, when there are more than two classes, this additional tie-breaker can be beneficial.

  After the training period in which the CLN, OCN, and CCN hyperboxes are made, the network is ready to predict on test samples. Like the standard Fuzzy Network, the membership value is calculated with respect to each CLN, and the max values with respect to each class are the accepted output; however, the prediction also takes into account the outputs of the OCN and CCN hyperboxes. The min value of each class from the OCN and CCN hyperbox outputs are considered as their predictions. These values, respective to each class, are then added to the values output from the CLN hyperboxes and the max value of the combined outputs is considered the network's prediction.

Thoughts about the network

  So is the GRFMMN my new go-to network for my modeling needs? Sadly, that is not the case. The performance of the network at its time of release was very competitive, but with the growth of Deep Learning, this older model just can't keep up. I tried it on a variety of data sets and found some weaknesses in its abilities. I noticed that this network underperforms in imbalanced data sets. I also noticed that the test time is rather lengthy for its performance. The code certainly can be optimized for faster performance yet, even with increased speeds, I don't see the payoff for the limited performance in testing. At this time, the field of fuzzy logic isn't making huge strides yet, time will tell.
