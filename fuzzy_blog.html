Python implementation of the paper "A General Reflex Fuzzy Min-Max Neural Network."

Intro

  A common issue that arises in creating models is deciding between high accuracy and low interpretability or high interpretability and low accuracy. This issue becomes even cloudier when the data doesn't fully represent the events that are being modeled. Due to the lack of information provided in the data, a compromise that allows for the continual use of the data is weakening the restrictions of the accepted outputs of the model. A modern implementation of this type of model is a Bayesian Neural Network. This model has weights that consist of two trainable parameters that represent mean and variance. This allows for the model to output a range of possible outcomes. Although Bayesian Networks and Bayesian Deep Learing overall is currently growing, there exisit another family of models that allow for high levels of interpretability. These models are called Fuzzy Networks.

Fuzzy Networks

  Fuzzy Networks are based around the theorms of fuzzy sets. Fuzzy sets were made by Lotfi A. Zadeh and Dieter Klaua in 1965. A fuzzy set consists of a set and a membership function which states how much an element(data-sample) is part of that particular fuzzy set. These ideas of fuzzy sets were futher developed by Zadeh into the ideas of fuzzy logic. Unlike classic logic, fuzzy logic allows for continous-intervals of truth. The continous truth intervals allow for an element to be given a a non-binary value from zero to one rating how much that element is part of a particular fuzzy set. Fuzzy sets also have very clean geometric properties allowing for easy visual intrpreation on how a particular element relates to a certain fuzzy set. Throught the rules of fuzzy logic, Fuzzy Networks were inspired.

  Fuzzy networks are usually classification models however, there are varients that do regression. In this post I will be covering Fuzzy Networks that do clasification. Furthermore, Fuzzy Networks are supervised learning models. The data set consits of training examples with labled targets of two or more different classes. Fuzzy Networks create trainable sets based around fuzzy sets called called hyperboxes. Hyperboxes conisit of two parameters, a min and a max, that define a lower and upper corner of a rectangular area. Although I am describing the hyperboxes having a rectangular area based on the min and max parameters, this is only for the sake of conceptual explination. Hyperbox min and max parameters can be vectors larges than two dimensions. Two dimensions simply allows for a convient visual explanation for the system. All the principles of the Fuzzy Network apply to N-dimension data samples. Hyperboxes also have a membership function that states how much an element is part of that particular hyperbox based on a its distance from the defined hyperboxe's area.

The membership function takes in several variables. The function takes the min~V and max~V parameter of the hyperbox. It takes in a tuning parameter gamma (gamma values vary from one to four on average) which adjust the sensitivity of the distance measurement. The membership function has a composed function within it, a ramp function, which sets a tested elements membership value to one if the element falls directly within it's area.

Finally, beyond the membership function, the hyperbox has tag saying which class it is associated to. Fuzzy Networks, during their training period, have two stages: expanison and contraction.

  During the expansion phase, hyperboxes are created or expanded by conecting to other hyperboxes. When a data sample is introduced during the expansion phase two outcomes can happen. One, a new hyperbox could be formed. This occurs when no hyperboxes meet an expansion critera (refer to paper for further detail), thus a new hyperbox of the respective labeled class is created. This implies whenever a data sampel from a class is introduced for the first time, a hyperbox with the tag of that respective class will always be made. The min and max parameters wil be a vector equiviliant to the training sample's values. It is neccesary that your input data be scalled between zero and one. Intially, min and max parameters are equal to each other, although this set is called a hyperbox, it initally, geometrically appears as a point. The second event that can occur during the expansion phase is expansion itself.

  If two points meet the expansion critera, then two hyperboxes join togther to form one large hyperbox with min and max parameters based on the min[min-parameter] and max[max-parameter] of the two sets. Expansion can only occur if the class of the two hyperboxes are the same. The second part of the training phase, contraction, occurs if the hyperbox that was just expanded, now overlaps an already existing hyperbox of different classes. Based off of of different cases in how the boxes overlap, the overlapping boxes get assigned new min and max parameter, causing the boxes to be no longer overlapping.

  After the training period in which hyperboxes are made, for the different data classes presented in the training data, the model is now ready to predict on test data. When the model is shown a test sample, the memebership value is calculated in respect to every hyperbox that was created. Of these calculated memebership values, the max memebership values from any particular defined class is considered the prediction.

  This is traditionally how Fuzzy Networks work however, in the paper "A General Reflex Fuzzy Min-Max Neural Network" by Abhijeet V. Nandedkar and Prabir Kumar Biswas, the idea of a Fuzzy Network was built further to create a new network called the General Reflex Fuzzy Min-Max Neural Network [1].

General Reflex Fuzzy Min-Max Neural Network

  The General Reflex Fuzzy Min-Max Neural Network, GFRMMN, is unique in the way in which it adds two new types of hyperboxes that help break ties between samples that are closely categorized. In total there are three types of hyperboxes in the GFRMMN: clasfying, overlap compensation, and containment compensation hyperboxes. The classfying hyperbox is the same hyperbox in the standard Fuzzy Network, so I will be discussing the two unique hyperboxes in the GFRMMN which are the overlap compensation neuron (OCN) and the containment compensation neuron (CCN).

  The overlap compensation neuron is created during the contracion phase of the Fuzzy Network's training. When the network contracts two overlapping hyperboxes, a new hyperbox, the overlap compensation neuron, is formed with the min and max values being assigned to the overlapping rectangular area of the two intial hyperboxes.

  Like the standard hyperbox, the OCN has its own membership function expressing how much an element is part of that particular hyperbox. The unique thing about the OCN is that it belongs to both classes that were overlapping. Although trivial sounding, when there are more than two classes, this additional tie breaker can be benefical.

  After the training period in which the CLN, OCN, and CCN hyperboxes are made, the network is ready to predict on test samples. Like the standard Fuzzy Network, the membership value is calculated in respect to each CLN and the max values in respect to each class are the accepted output however, the prediction also takes into account the outputs of the OCN and CCN hyperboxes. The min value of each class from the OCN and CCN hyperbox outuputs are considered as their predictions. These values, respective to each class, are then added to the values outputed from the CLN hyperboxes and the max value of the combined outputs is considered the network's prediction.
